# Mandatory part
The project content is up to you. Yes, you have to bring your own ideas and decide together what application to build as a team.
โปรเจคแล้วแต่เราว่า ว่า จะพัฒน่าเว็บ ไปในทิศทางไหนดี

It's a step forward from what you did previously in the Common Core. You need to think about the project as a whole, not just the features you're going to implement.
สรุปง่ายๆว่า ต้องคิดว่าโปรเจคนี้มาปรับในโปรเจคนี้ ไม่ใช่สักแต่จะทำ features โดยไม่นำสิ่งที่เรียนมาปรับ

of course, you won't be completely free---there are constraints---but the idea is yours.
แน่นอนว่าจะอิสระออกแบบ 100% ต้องมีข้อจำกัดบางอย่าง(ที่เขาว่าน่าจะเกี่ยวกับฟีเจอร์ที่ต้องมีแหละในเว็บ) แต่ไอเดียว่าเว็บไซค์ทำออกมาแบบไหนอยู่ที่เราเลย

# III.1 What are we doing?
First, you will have to create a comprehensive README.md file. The detailed requirements for the README are specified in the README Requirements section at the end of this document.
อย่างแรกจะต้องสร้างไฟล์ README.md รายละเอียดที่ต้องการสำหรับ README ตามฟอ์ม ใน หน้าท้ายๆของ subject pdf แล้ว


## Project Examples: Your project can take many form. here are some  valid examples:
- A multiplayer Pong game with tournament system (เกมปองที่มีระบบแข่งขัน ทัวนาเม้น)
- A collaborative platform with real-time features(แพลตฟอร์มที่ใช้ในการ โควงานกัน นึกไม่ออกก็เช่น slack, microsoft team etc.)
- A online game (Chess, Tic-Tac-Toe, etc.) with matchmaking (เกมออนไลน์ เช่น หมากรุก, เกมเอ็กซ์โอ และอื่นๆ )
- A project management application (เว็บแอพที่ไว้จัดการลำดับโปรเจค เช่น notion, trello, miro etc.)
- Any other creative web application that meets the requirements (เว็บแอพอื่นๆที่มันจะเกี่ยวข้องตาม requirement ของ subject)

The key is to create something engaging that demonstrates your technical skills and creativity.
จุดหลักๆคือการสร้างบางอย่างที่สามารถจำลองสาธิตในการดูทักษะของtechnical และ creativity ของเรา
