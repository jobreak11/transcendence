# IV.2 Accessibility and Internationalization
- Major: Complete accessibility compliance (WCAG(Web Content Accessibility Guidelines) 2.1 AA) with screen reader support, keyboard navigation, and assistive technologies.(ผู้พิการสามารถใช้งานเว็บได้ เพราะงั้น จะต้องออกแบบ ให้เป็นมิตรกับ ผู้พิการ เช่น keyboard Navigation(ใช้งานคีย์บอร์ดได้แม้ไม่มีเมาส์), Scrren Reader Support(มีระบบอ่านให้สำหรับตาบอดมองไม่เห็น), Assistive Technologies(เกี่ยวกับเรื่องของ text สีต่างๆ ความคมชัด ที่อำนวยให้คนตาบอดสี))
- Minor: Support for multiple languages (at least 3 languages). (รองรับภาษาหลายภาษา อย่่างน้้อย 3 ภาษาขึ้นไป)
    - Implement i18n (internationalization) system. (ห้ามสร้างเเว็บแยกตามภาษาต่างๆให้ใช้ library มาตรฐานของภาษานั้นๆ เช่น React ฬช้ react-i18next)
    - At least 3 complete language translations. (ทุกหน้าเมื่อ switch ภาษา แล้ว เช่น "complete" ต้องแปลในทุกๆหน้า ไม่ใช่แปลหน้าใดหน้านึง)
    - Language switcher in the UI. (มีปุ่มสลับภาษาบนหน้าเว็บ)
    - All user-facing text must be translatable. (๘้อความทั้งหมดที่ผู้ใช้เห็นต้องแปลได้ เช่น button จะเก็บjson ไฟล์ที่เมื่อสลับภาษา จะเก็บ ภาษาต่างๆ เช่น class นี้ มี en.json th.json ที่เก็บปุ่มภาษาต่างๆของ ปุ่ม login ไว้)
- Minor: Right-to-left (RTL) language support. (รับภาษาที่แปลจากขวาไปซ้าบ)
    - Support for at least one RTL language (Arabic, Hebrew, etc.). (รองรับภาษา่ที่แปลจากขวาไปซ้ายอย่างน้อย 1 ภาษา เช่น Arabic, Hebrew และ อื่นๆ)
    - Complete layout mirroring (not just text direction). (โครงสร้างtext และอื่นๆ ต้องกลับทิศทางด้วย)
    - RTL-specific UI adjustments where needed. ()
    - Seamless switching between LTR and RTL. (มีโหมดเปลี่ยน LTR(ซ้ายไปขวา) และ RTL(ขวาไปซ้าย) ต้องไม่มีปัญหา)
