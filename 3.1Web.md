# IV.1 Web
Major = 2 points, Minor = 1 point
-Major: Use a framework for both the frontend and backend. (ใช้ framework ทั้ง frontend และ backend) \
  - Minor: Frontend framework (React, Vue, Angular, Svelte, etc.). \
  - Minor: Backend framework (Express, NestJS, Django, Flask, Ruby on Rails, etc.). \
  - Major: Full-stack frameworks (Next.js, Nuxt.js, SvelteKit) count as both if you use both their frontend and backend capabilities. (framework ที่เป็น Full-stack จะถูกนับเป็น 2 คะแนน เพราะเป็นทั้ง frontend และ backend ในตัวเลย) \
-Major: implement real-time features using WebSockets or similar technology. (ใช้ ฟีเจอร์ real-time เช่น WebSockets หรือ อื่นๆ) \
  - Real-time updates across clients. (อัพเดทข้อมูลทันทีจาก client) \
  - Handle connection/disconnection gracefully. (ไม่มีปัญหาในการ connection/disconnection ของ client) \
  - Efficient message broadcasting. (ข้อความ broadcasting(ประกาศ) ไม่มีปัญหา) \
-Major: Allow users to interact with other users. The minumum requirements are (ฟีเจอร์พื้นฐานที่ User ใช้งานกับ User คนอื่นๆ): \
  - A basic chat system (send/receive messages between users). (ระบบส่งข้อความ) \
  - A profile system (view user information). (ระบข้อมูลส่วนตัว) \
  - A friends system (add/remove friends. see friends list). (ระบบเพื่อน เพิ่ม,ลบ เพื่อน และ ดูรายชื่อเพื่อน) \
-Major: A public API to interact with the database with a secured API key, rate limiting, documentation, and at least 5 endpoints (จะต้องทำ Public API เพื่อให้ช่องทางอืนๆสามารถใช้งานระบบฐานข้อมูลของเราได้ ซึ่งจะต้องมี Secured API key(ระบบกุญแจความปลอดภัย), Rate Limiting(จำกัด เวลา, จำนวนครั้ง ในการใช้งาน), Documentation(เอกสารคู่มือการใช้งาน) ต้แงมีอย่างน้อย 5 ช่องทางการเชื่อมต่อ): \
    - GET /api/{something} (สำหรับ "ดึงข้อมูล" เช่น GET /api/users ดูรายชื่อผู้ใช้ หรือ GET /api/scores เพื่อดูคะแนนรวม) \
    - POST /api/{something} (สำหรับ "สร้างข้อมูลใหม่" เช่น POST /api/match เพื่อบันทึกผลการแข่งขันใหม่เข้าไป) \
    - PUT /api/{something} (สำหรับ "แก้ไข/อัปเดตข้อมูลเดิม" เช่น PUT /api/user/profile เพื่อแก้ไขโปรไฟล์) \
    - DELETE /api/{something} (สำหรับ "ลบข้อมูล" เช่น DELETE /api/tournament/{id} เพื่อลบทัวร์นาเมนต์ที่ยกเลิก) \
-Minor: Use an ORM for the database.(ใช้ ORM หรือ การเขียน database แบบ object ด้วย)
-Minor: A complete notification system for all creation, update, and deletion actions.(มีระบบแจ้งเตือน creation(เช่น มีผู้ใช้เข้าสู่ระบบ etc.), update(มีการอัพเดทข้อมูลใหม่ เช่น user มีการเปลี่ยนชื่อเกิดขึ้น etc.), Deletion(เช่น การที่แอดมินลบผู้ใช้ออกจากระบบ etc.)) \
-Minor: Real-time collaborative features (shared workspaces, live editing, collborative drawing, etc.). (มีฟีเจอร์ในการทำงานร่วมกัน เช่น shared workpaces, live editing, collaborative drawing) \
-Minor: Progressive Web App (PWA) with offline support and installability.(offline support คือเว็บต้องไม่เป็นหน้าไดโนเสาร์ จะบันทึกข้อมูลจากเว็บที่เราเปิดไว้ล่าสุด และ installiability คือ เว็บต้องสร้าง shortcut ที่หน้าจอคอม หรือ สร้างapp ไว้หน้าขอมือถือได้) \
-Minor: Custom-made design system with reusable components, including a proper color palette, typography, and icons (minimum: 10 reusable components).(ปรับแต่ง พวกสิ่งต่างๆที่อยู่หน้าเว็บ เช่น ปุ่ม, หน้าต่าง ตีม หรือื่นๆ อย่างน้อย 10ชิ้น เอาง่ายๆ คือไม่ควรเอา พวกของที่ทำสำเร็จแล้วมาแปะหน้าเว็บของเรา) \
-Minor: Implement advanced search functionality with filters, sorting, and pagination.(มีระบบการค้นหา มี filters(เลือกบางส่วน), sorting(จัดเรียงลำดับใหม่) และ pagination(แบ่งหน้า)) \
-Minor: File upload and management system(ระบบ อัพโหลด และ จัดการ ไฟล์) \
  -Support multiple file types (images, documents, etc.). (รองรับได้หลายๆประเภทไฟล์) \
  -Client-side and server-side validation (type, size, format). (มีการเช็คสองชั้น รูปแบบ, ขนาด, ประเภท ทั้งฝั่งของ client และ server) \
  -Secure file storage with proper access control.(จัดเก็บความปลอดภัยและคุมสิทธิ์ เช่น Access controlไฟล์บางประเภท จำกัดคนดู, Secure Storage ห้ามใช้ชื่อซ้ำในการจัดเก็บไฟล์) \
  -File preview functionality where applicable (ต้องแสดงตัวอย่างไฟล์ให้ user เห็นทันที เช่น ถ้าอัพโหลดรูปผ่านแล้ว ต้องขึ้นรูป Thumbnail เล็กๆ โชว์บนหน้าจอ หรือถ้าอัพโหลดไฟล์ PDF ก็ต้องแสดงชื่อไฟล์ ขนาด และไอคอนเอกสารอย่างสวยงาม) \
  -Progress indicators for uploads(มี % การอัพโหลด ว่า อัพโหลดไฟล์นี้เสร็จไปกี่ % 0% ถึง 100%) \
  -Ability to delete uploaded files.(เจ้าของไฟล์ต้องมีปุ่ม "ลบไฟล์" ลบทั้ง โฟลเดอร์เก็บข้อมูลในเซิร์ฟเวอร์ และ ลบชื่อออกจากฐานข้อมูลด้วย) \
