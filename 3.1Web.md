# IV.1 Web
Major = 2 points, Minor = 1 point
-Major: Use a framework for both the frontend and backend. (ใช้ framework ทั้ง frontend และ backend)
  -Minor: Frontend framework (React, Vue, Angular, Svelte, etc.).
  -Minor: Backend framework (Express, NestJS, Django, Flask, Ruby on Rails, etc.).
  -Major: Full-stack frameworks (Next.js, Nuxt.js, SvelteKit) count as both if you use both their frontend and backend capabilities. (framework ที่เป็น Full-stack จะถูกนับเป็น 2 คะแนน เพราะเป็นทั้ง frontend และ backend ในตัวเลย)
-Major: implement real-time features using WebSockets or similar technology. (ใช้ ฟีเจอร์ real-time เช่น WebSockets หรือ อื่นๆ)
  -Real-time updates across clients. (อัพเดทข้อมูลทันทีจาก client)
  -Handle connection/disconnection gracefully. (ไม่มีปัญหาในการ connection/disconnection ของ client)
  -Efficient message broadcasting. (ข้อความ broadcasting(ประกาศ) ไม่มีปัญหา)
-Major: Allow users to interact with other users. The minumum requirements are (ฟีเจอร์พื้นฐานที่ User ใช้งานกับ User คนอื่นๆ):
  - A basic chat system (send/receive messages between users). (ระบบส่งข้อความ)
  - A profile system (view user information). (ระบข้อมูลส่วนตัว)
  - A friends system (add/remove friends. see friends list). (ระบบเพื่อน เพิ่ม,ลบ เพื่อน และ ดูรายชื่อเพื่อน)
