# IV.3 User Management
Major = 2 points, Minor = 1 point
- Major: Standard user management and authentication (มาตรฐานของ ระบบจัดการ user และ สิทธิ์ต่างๆของ user)
  - User can update their profile information. (user อัพเดท ข้อมูลส่วนตัวได้)
  - User can upload an avatar (with a default avatar if none provided). (user อัพโหลด avatar ของตัวเองได้ เช่น รูปโปรไฟล์ ถ้าไม่มี รูป avatar อัพโหลด ต้องมี รูป avatar default ไว้ด้วย)
  - User can add other users as friends and see their online status. (user สามารถเพิ่มเพื่อน และดู status ว่าเพื่อนออนไลน์ไหม ***อันนี้มีเอี่ยว3.1Web ใช้ search ctrl+f ได้ อยู่ใน  "Major: Allow users to interact with other users. The minimum requirements are")
  - User have a profile page displaying their information (โชว์หน้าข้อมูลส่วนตัวของ user)
- Minor: Game statistics and match history (requires a game module) (หน้าที่เก็บสถิติเกม และ ประวัติ การแข่ง match ก่อนหน้า ของ user ***ข้อนี้ เอี่ยว game module)
  - Track user game statistics (wins, loses, ranking, level, etc.). (มีหน้าดูประวัติ user เช่น ชนะกี่ครั้ง, แพ้กี่ครั้ง, อยู่ที่ระดับอะไร, เลเวลอะไร และ อื่นๆ)
  - Display match history (1v1 games, dates, results, opponents.). (มีประวัติ match แต่ละรอบ เช่น 1v1 games,  วันที่แข่ง, ผลว่าชนะหรือแพ้ในรอบนั้น, ฝั่งตรงข้ามเป็นใคร)
  - show achievements and progression. (มีระบบ ความสำเร็จ เหมือนเหรียญตตรา เช่น ชนะ ติดต่อกัน 10 รอบ ปลดล็อค "ผู้ชนะทองคำ")
  - Leaderboard integration. (เหมือนสถิติทั้ง server ของเว็บว่า ใครชนะ ติดต่อกันมากที่สุด leaderboard จะเก็บสถิติ จำนวนชนะ ทุกๆคน และจัดอันดับ ทั้ง server)
## ****** This module requires you to have implemented at least one game (see "Gaming user experience" section). You cannot claim this module without a funcrtional game.) (เอาง่ายๆ module อันบนนี้ ต้องทำ module หลักที่เป็น เกมก่อน ถึงจะทำ module อันบนนี้ได้)
- Minor Implement remote authentication with OAuth 2.0 (Google, GitHub, 42, etc.). (สามารถล็อคอิน ผ่านระบบภายนอกได้ และการยืนยันตัวตน จะต้องถูกผลักไปที่ google, GitHub, 42 etc. เอง)
- Major: Advanced permissions system(มีการจัดการระบบสิทธิ์อณุญาตเข้าถึงในระดับสูง):
  - View, edit, and delete users (CRUD). (จัดการ ดูได้อย่างเดียว หรือ ทั้งดูและปรับแต่งได้ แม้กระทั่ง ลบ user คนนั้นออกจากลุ่มได้)
  - Roles management (admin, user, guest, moderator, etc.). (มีระบบจัดการยศ เช่น admin, user, guest, moderator และ อื่นๆ)
  - Different views and actions based on user role. (สิ่งที่ทำได้และไม่ได้ ดูได้และไม่ได้ ขึ้นอยู่กับ role ของ user นั้นๆ เช่น post นี้ ดูได้แค่ admin เท่านั้น user ทั่วไปดูไม่ได้)
- Major: An organization system(มีระบบการจัดการตำแหน่งหรือกลุ่มต่างๆยศทั้งหมด ดังนี้):
  - Create, edit, and delete organizations. (สามารถสร้าง ปรับแต่ง หรือ ลบ ตำแหน่ง/กลุ่ม/ยศ นั้นๆได้)
  - Add users to organizations. (เพิ่ม user เข้า กลุ่ม/องค์กร ได้)
  - Remove users from organizations. (ลบ user ออกจาก กลุ่ม/องค์กร ได้)
  - View organizations and allow users to perform specific actions within an organization (minimum: create, read, update). (กลุ่ม/องค์กร ให้ ผู้ใช้ สิทธิ์ ที่่จะทำบางอย่างได้อย่างจำกัด ในข้อ นี้ ขั้นต่ำ แนะนำว่า ต้อง สร้าง, อ่าน, และ อัพเดทได้)
- Minor: Implement a complete 2FA (Two-Factor Authentication) system for the users. (ใช้ระบบการยืนยันตัวตน สองขั้น นอกจาก login ปกติแล้วต้องมีการยืนยันตัวตนแบบล็อคอินอีกขั้น เช่น ใช้ OTP ในการล็อคอินอีกขั้น)
- Minor: User activity analytics and insights dashboard. (หน้าวิเคราะห์ คนเข้าออกเว็บไซค์ เก็บไว้ว่าหน้าไหนคนเข้าไปทำอะไร และ หน้า insights dashboard ที่เก็บจำนวนคนเข้า ตามช่วงเวลาต่างๆ)
