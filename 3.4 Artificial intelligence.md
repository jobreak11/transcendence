# IV.4 Artificial Intelligence
Major = 2 points, Minor = 1 point
- Major: Introduce an AI Opponent for games. (บอทของเกม)
  - The AI must be challenging and able to win occasionally. (ตัวบอทต้องท้าทาย เก่งพอที่จะชนะ user ได้ บ้าง)
  - The AI should simulate human-like behavior (not perfect play). (ทำบอทให้ ไม่เก่งเกินไป มี โอกาสผิดพลาดได้, ให้ บอท AI มีเวลา ในการตัดสินใจเหมือน user ไม่ใช่ว่า user ทำแบบนี้แล้ว บอท AI แก้ได้เลยทันที)
  - If you implement game Customization options, the AI must be able to use them. (ถ้าในโหมดปรับแต่งตัวเกม user ทำได้ ตัว บอทAI เองก็ทำได้เหมือนกัน)
  - You must be able to explain your AI implementation during evaluation. (อธิบายได้ว่า ตัว บอท AI มันทำงานยังไง ในช่วง evaluation)
## ****** This module requires you to have implemented at least one game see "Gaming user experience" section). The AI must be able to play your game competently. (เอาง่ายๆ module อันบนนี้ ต้องทำ module หลักที่เป็น เกมก่อน ถึงจะทำ module อันบนนี้ได้ และ ตัว บอท AI สามารถเล่นเกมเราได้แบบไม่มีปัญหาอะไร)
- Major: Implement a complete RAG (Retrieval-Augmented Generation) system. (AI จะดึงข้อมูลแหล่งที่มาแทนที่จะสร้างคำตอบเอง)
  - Interact with a large dataset of information (AI จะมี ข้อมูลขนาดใหญ่เตรียมไว้ก่อนจะตอบเรา เช่น คู่มือเว็บ, FAQ ของเว็บ และอื่นๆ)
  - Users can ask questions and get relevant answers (เมื่อ user ถาม ต้องตอบในประเด็นที่สิ่งที่ user ต้องการจะรู้จริงๆ ยึดตาม Dataset ที่มีอยู่)
  - Implement proper context retrieval and response generation. (AI เรา ดึง context คำถามจาก user หลัลงจากดึงมาแล้วต้องพึ่งพวก LLM เช่นใช้ OpenAI ให้มันสร้างคำตอบให้)
- Major: Implement a complete LLM system interface
  - Generate text and/or images based on user input. (สร้าง ข้อความ หรือ รูปภาพ ตามสิ่งที่ผู้ใช้กรอกเข้ามา โดยโยนให้ OpenAI สร้าง text หรือ รูปให้ได้)
  - Handle streaming responses properly. (การเจนคำตอบยาวๆ ต้องไม่ใช่แบบส่างทีเดียวทั้งก่อน ต้องส่งหาผู้ใช้ทีละตัวแบบ realtime ถ้านึกไม่ออกก็ minitalk ที่เคยทำ เพื่อให้ ผู้ใช้รู้สึกว่าเว็บไม่ค้างตอนที่ AI พิมพ์แล้ว)
  - Implement error handling and rate limiting. (เวลาเกิด error เว็บค้าง,เน็ตหลุด,ผู้ใช้พิมพ์คำไม่เหมาะสม ต้องจัดการได้ทั้งหมด)
- Major: Recommendation system using machine learning. (ระบบให้คำแนะนำ user)
  - Personalized recommendations based on behavior. (แนะนำสิ่งต่างๆให้ user ตาม พฤติกรรมของ user เช่น ถ้ามี TOP 5 ยอดนิยม แต่คนนั้นไม่ชอบ แต่ AI เอามาแนะนำ ถือว่า ไม่ถูกต้องเพราะ ไลฟ์สไตล์ของ คนทั่วไป ไม่ใช่ ของ user)
  - Collaborative filtering or content-based filtering. (แนะนำตามความตล้ายของกลุ่มผู้ใช้ เช่น เอาจากรสนิยมคนที่คล้ายๆกัน หรือ แนะนำตามคุณลักษณะของสิ่งนั้น เช่น ถ้าชอบอันนี้ น่าจะชอบอย่างอื่นที่คล้ายๆกัน)
  - Continuously improve recommendations over time. (โมเดล AI ที่เรามีต้องเรียนรู้เรื่อยๆ ไม่ใช่ fixed mindset ไม่เรียนรู้อะไรเลย)
- Minor: Voice/speeds integration for accessibility or interaction. (ให้ AI ตอบโต้ด้วยเสียงได้ เพื่อความสะดวกในการใช้งาน บวกกับ คนพิเศษที่มองไม่เห็นก็ใช้งานได้)
- Minor: Sentiment analysis for user-generated content. (AI วิเคราะห์ความรู้สึกผ่าน text ที่ user prompt ได้)
- Minor: Image recognition and tagging system. (AI จดจำรูปภาพ และให้ tag รูปภาพนั้นๆ)
