# IV.7 Devops
Major = 2 points, Minor = 1 point
- Major: Infrastructure for log management using ELK (Elasticsearch, Logstash, Kibana). (ระบบ Log ของระบ เว็บทั้งหมด)
  - Elasticsearch to store and index logs. (ข้อมูล index ที่เก็บ log ของ user หรือ อื่นๆ ที่ส่งเข้ามา)
  - Logstash to collect and transform logs. (ท่อไหลข้อมูล ดักรับ Log ต่างๆ เอามา ตัดจำ/จัดโครง parsing & transforming ก่อนส่งไปที่ Elasticsearch)
  - Kibana for visualization and dashboards. (Kibana สำหรับสร้างภาพ visual และทำ Dashboard แสดงผล จาก Elasticsearch ให้ดูภาพได้อย่างง่ายกว่า อะไรบ้างที่ส่งมาที่เว็บเรา เช่น แสดง กราฟจำนวน HTTP 500 error, จำนวนผู้ใช้ตามช่วงเวลาต่างๆ)
  - Implement log retention and archiving policies. (กำหนดนโยบายการจัดเก็บและสำรองข้อมูล Log)
  - Secure access to all components. (ป้องกันและรักษาความปลอดภัยในการเข้าถึงทุกๆ Component)
- Major: Monitoring system with Prometheus and Grafana. (ระบบเฝ่้าระวังและติดตามประสิทธิภาพ ด้วย Prometheus และ Grafana)
  -  Set up Prometheus to collect metrics. (ติดตั้งและตั้งค่า Prometheus เพื่อเก็บรวบรวมข้อมูล Metrics ที่คอยดึงค่าต่างๆ เช่น การใช้ CPU, RAM, จำนวน Request, latency จาก services ต่างๆ)
  -  Configure exporters and integrations. (ตั้งค่า exporters เชื่อมกับระบบต่างๆ เช่น สถานะระบบปฏิบัติการ CPU, RAM, DISK, Network เชื่อมกับ สถานะ Docker Containers)
  -  Create custom Grafana dashboards. (สร้าง Dashboard ทืี่กำหนดเอง ดึงผลลัพธ์จาก Prometheus มาวาดเป็นกราฟให้ดูง่าย)
  -  Set up alerting rules. (ตั้งค่าการแจ้งเตือนเช่น ถ้า CPU กินพื้นที่สูงเกิน 80% ต้องบอกว่า ตอนนี้ขั้นวิกฤตแล้วต้องทำอะไรสักอย่างกับเว็บไซค์)
  -  Secure access to Grafana. (ป้องกันและเพิ่มความปลอดภัยในการเข้าถึง Grafana เช่น ตั้งค่าระบบยืนยันตัวตอน, ปิดการเปิดให้คนนอก(Anonymous Users)เข้าหน้า dashboardโดยไม่ล็อกอิน)
- Major: Backend as microservices. (ฺระบบหลังบ้าน microservices ของ backend)
  - Design loosely coupled services with clear interfaces. (ออกแบบบริการย่อยที่มีความผูกผันกันต่ำ และมี หน้าตาการเชื่อมต่อที่ชัดเจน เอาง่ายๆ service แยกระบบกันเหมือน docker container ชัดเจน ไม่ต้องพึ่งพาโค้ด จากที่อื่นๆ ซึ่งจะคุยกันได้ต้องใช้ API Endpoint เท่านั้นการที่ใช้แบบนี้เพื่อที่ service ตัวใดตัวนึงพัง จะไม่กระทบกับตัวอื่นๆ)
  - Use REST APIs or message queues for communications. (ใช้ REST APIs หรือระบบคิวข้อความ ในการสื่อสารระหว่างกัน Synchronous (REST APIs) service A ยิง HTTP/HTTPS Request (GET,POST, ฯลฯ) ไปขอข้อมูลจาก service b โดยตรง)
  - Each service should have a single responsibility. (แต่ละ service ทำหน้าที่อย่างเดียว)
- Minor: Health and status page system with automated backups and disaster recovery procedures. (ระบบเช็คระบบของเว็บปัจจุบัน หน้าแสดงสถานะ และ ระบบสำรองข้อมูลอัตโนมัติพร้อมแผนกู้คืนเวลาเกิดเหตุฉุกเฉิน)
