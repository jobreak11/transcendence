# IV.6 Gaming and user experience
Major = 2 points, Minor = 1 point
- Major: Implement a complete web-based game where users can play against each other. (เป็นเกมที่แข่งขันกับฝั่งตรงข้าม)
  - The game can be real-time multiplayer (e.g., Pong, Chess, Tic-Tac-Toe, Card games, etc.). (เกมที่เป็นการแข่งขัน สดๆ แมชต่อแมช เช่น ปอง, หมากรุก, เกมxo, เกมการ์ด และ อื่นๆ)
  - Players must be able to play live matches. (ผู้เล่นสามารถเล่นแมตส์สดๆได้)
  - The game must have clear rules and win/loss conditions. (เกมมีกฎ ชนะ/แพ้ ที่ชัดเจน)
  - The game can be 2D or 3D. (เกมสามารถเป็นได้ทั้ง 2D หรือ 3D เลือกอย่างใดอย่างนึง)
- Major: Remote players - Enable two players on separate computers to play the same game in real-time (เล่นด้วยกันเล่นแบบแยกเครื่องกันได้่)
  -  Handle network latency and disconnections gracefully. (จัดการความหน่วง และ ตัดการเชื่อมต่อ ของผกู้เล่นที่เล่นด้วยกัน ได้ไม่มีปัญหาอะไร)
  -  Provide a smooth user experience for remote gameplay. (ระหว่างที่ผู้เล่น เล่นด้วยกัน ต้องไม่กระตุกเล่นลื่น)
  -  Implement reconnection logic. (ให้มีระบบ เชื่อมต่อใหม่อีกครั้งถ้าเน็ตหลุด หรือเกิดปัญหาบางอย่าง)
- Major: Multiplayer game (more than two players). (เกมรองรับผู้เล่นมากกว่า 2 คนขึ้นไป)
  - Support for three or more players simultaneously. (รองรับผู้เล่น 3 คน หรือมากกว่านั้น พร้อมกันได้)
  - Fair gameplay mechanics for all participants. (รูปแบบการเล่นของแต่ละคนต้องเท่าเทียมกัน)
  - Proper synchronization across all clients. (ระหว่างโหมดผู้เล่นหลายคนต้องไม่มีปัญหาอะไรสามรารถเชื่อมต่อกันได้ลื่นไหล)
## ****** This module requires you to have implement at least one game (see "Gaming and user experience" section). You’re extending your game to support three or more players. (โมดูล อันบนนี้ จะถูกนับเมื่อเกมรองรับผู้เล่น สามคน ขึ้นไป)
- Major: Add another game with user history and matchmaking. (เพิ่มเกมที่สอง จากเกมหลัก พร้อมมีประวัติ user และ ระบบจับคู่)
  - Implement a second distinct game. (สร้างเกมที่แตกต่างจากเกมแรกอย่างชัดเจน)
  - Track user history and statistics for this game. (มีระบบบันทึกผลของเกม และ สถิติ ของ user)
  - Implement a matchmaking system. (สร้างระบบ จับคู่หาคู่แข่งให้ผู้เล่น)
  - Maintain performance and responsiveness. (ทำให้ระบบไม่แลค ไม่ช้า ไม่อืด ไม่สะดุด ลื่นไหล)
## ****** This module requires you to have implement at least one game (see "Gaming and user experience" section). You cannot claim this module without having a functional first game. (โมดูลอันนี้ จะถูกนับเมื่อเกมแรกทำงานอย่างถูกต้องไม่มีสะดุด)
- Major: Implement advanced 3D graphics using a library like Three.js and Babylon.js. (ใช้เทคโนโลยีเกม 3D โดยใช้ library เช่น Three.js หรือ Babylon.js)
  - Create an immersive 3D environment. (สร้างแวดล้อม 3D ในเกม)
  - Implement advanced rendering techniques. (หาเทคนิคในการ rendering(สร้างโมเดลภาพ3D/2D) ขั้นสูง)
  - Ensure smooth performance and user interaction. (ระบบลื่นไหล และ ผู้เล่น ตอบสนองทันที ไม่มีสะดุด)
- Minor: Advanced chat features (enhances the basic chat from 'User interaction' module). (ระบบแชทขั้นสูง (จากระบบแชทปกติ เพิ่ม ระบบ 'การโต้ต้อบของ user' โมดูล))
  - Ability to block users from messaging you. (สามารถ บล็อค ผู้เล่น คนอื่นที่ส่งข้อความมาได้)
  - Invite users to play games directly from chat. (เชิญผู้เล่นอื่นผ่านข้อความได้ เช่น เหมือน steam เชิญคยอ่ื่นเล่นเกมนี้ ผ่านกล่องข้อความแชท)
  - Game/tournament notifications in chat. (มีระบบแจ้งเตือน เกี่ยวกับ การแข่งขัน ทัวร์นาเม้นในแชท)
  - Access to user profiles from chat interface. (สามารถเข้าไปหน้าส่วนตัว ของผู้เล่นคนอื่นได้ ผ่านหน้าตากล่องข้อความแชท)
  - Chat history persistence. (ระบบแชท ต้องเก็บประวัติคุยกันด้วยทั้งหมด ไม่ใช่แชทเสร็จแล้วประวัติการคุยหายไป)
  - Typing indicators a nd read receipts. (ระบบแชทเมื่อผู้เล่นคนอื่นกำลังพิมพ์ต้องขึ้นว่าผู้เล่นคนนั้น "กำลังพิมพ์อยู่..." และ ถ้าเราอ่านข้อความที่ผู้เล่นคนอื่นส่งมาแล้ว ต้องขึ้นว่า "อ่านเมื่อ....")

(โมดูลบนนี้เอามาจาก 3.1 Web "Allow users to interact with other users." ถ้า basic chat จากโมดูลนั้นไม่เอามาใช้งานในโปรเจค โมดูลอันบนนี้จะใช้ไม่ได้)
- Minor: Implement a tournament system. (ระบบแข่งขันแบบทัวร์นาเม้น)
  - Clear matchup order and bracket system. (มีลำดับประกบคู่ชัดเจน และ มีตารางสายการแข่งขัน)
  - Track who plays against whom. (มีระบบันทึกว่า ผู้เล่นคนไหน แข่งกับ ผู้เล่นคนไหน)
  - Matchmaking system for tournament participants. (ระบบจัดหาคู่ผู้เล่นสำหรับผู้เข้าร่วมแข่งขัน)
  - Tournament registration and management. (มีระบบลงทะเบียนและการจัดการทัวร์นาเมนต์)
## ****** This module requires you to have implemented at least one game (see "Gaming and user experience" section). You cannot have tournaments without a game to play. (โมดูล บนนี้ ต้องสร้างเกมออกมาเล่นได้ก่อนจะมีระบบทัวร์นาเมนต์ เพราะทัวร์นาเมนต์เป็นระบบเสริมที่จะมีหรือไม่ก็ได้)
- Minor: Game customization options. (ระบบปรับแต่งความสามารถในเกม)
  - Power-ups, attacks, or special abilities. (เพิ่มพลัง, การโจมตี หรือ ความสามารถพิเศษอื่นๆ)
  - Different maps or themes. (มีแผนที่หรือตีมต่างๆ ที่แตกต่างกัน)
  - Customizable game settings. (ปรับโหมดเกมต่างๆได้ในห้องเกมที่จะเริ่มเกม)
  - Default options must be available. (ถ้าผู้เล่นสร้างห้องเริ่มเกม โดยไม่ปรับแต่งอะไร จะต้องมี default ให้ห้องเกมที่ผู้เล่นคนนั้นๆสร้างขึ้นมาเสมอ)
## ****** This module requires you to have implemented at least one game (see "Gaming and user experience" section). You're adding customization to an existing game. (โมดูล บนนี้ ต้องสร้างเกมออกมาเล่นได้ก่อน เพราะ ระบบปรับแต่งเกม เป็นฟังก์ชั่นเสริม ที่จะถูกเพิ่มในแนวเกมที่เราทำมาอยู่แล้ว)ฃ
- Minor: A gamification system to reward users for their actions. (ระบบขึ้นระดับผู้เล่น)
  - Implement at least 3 of the following: achievements, badges, leaderboards, XP/level system, daily challenges, rewards. (ใช้อย่างน้อย 3 ฟีเจอร์ตามนี้: ระบบปลดล็อคความสำเร็จ ตามเงื่อนไข เช่นชนะสามรอบติดต่อกันปลดล็อคความสำเร็จ "ผู้ไม่แพ้", ระบบป้ายเกียรตยศ, กระดานสถิติคะแนนโดยรวมทั้งหมดของผู้เล่นทั้งเซิฟ TOP1 ลงไปเรื่อยๆ, ระบบ XP/level หรือ ระบบค่าประสบการณ์ของผู้เล่นเพื่อเลื่อนระดับ, ภารกิจประจำวัน และ ระบบรางวัลตอบแทน)
  - System must be persistent (stored in database). (ข้อมูลทุกอย่างต้องยังถูกบันทึกไว้ใน database เช่น ถ้าผู้เล่นคนนั้นเวลmaxแล้ว ข้อมูลพวกนั้นต้องอยู่ตลอดไป เวลาผู้เล่นออกจากระบบ ไม่ใช่ออกปุ้บรีกลับเป็นเวล 1 ใหม่)
  - Visual feedback for users (notifications, progress bars, etc.) (มีหลอด 0-100% ให้ผู้เล่นเห็นว่า คืบหน้าถึงไหนแล้ว เช่น ความคืบหน้าของภารกิจประวัน, ระดับค่าประการณ์ กี่ % ถ้านึกไม่ออกก็ ระดับ % เลเวลของระบบ 42 นั่นแหละ)
  - Clear rules and progression mechanics. (กฎเกณฑ์ในการ ไต่ระดับ เทียบกับ ผลตอบแทนร่างวัลที่ได้ต้องสมเหตุสมผล เช่น ชนะ 1 แมตช์ = ได้ 100 xp ค่าประสบการณ์ ถ้า ชระ 2 แมตช์ = ได้ 200 xp ค่าประสบการณ์)
