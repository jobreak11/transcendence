# Readme Requirements
A README.md file must be provided at the root of your Git repository. Its purpose is to allow anyone unfamiliar with the project (peers, staff, recruiters, etc.) to quickly understand what the project is about, how to run it, and where to find more information on the topic. (เอาไฟล์ไว้ที่ root สั้นๆเลย)
The README.md must include at least: (ข้างล่างนี้จะบอกว่า ใน README.md ต้องมีอะไรบ้าง)
- The very first line must be italicized and read: This project has been created as part of the 42 curriculum by <login1>[, <login2>[, <login3>[...]]]. (Header ต้องเป็นฟอร์ม "This project has been created as part of the 42 curriculum by <login1>[, <login2>[, <login3>[...]]]")
- A "Description" section that clearly presents the project, including its goal and brief overview. (หัวข้อ "Descrption" จะต้องอธิบายคร่าวๆ แต่ชัดเจน ให้เห็นถึงเป้าหมาย กับภาพรวมย่อๆ)
- An "Instructions" section containing any relevant information about compilation, installation, and/or execution. (หัวข้อ "Instruction" จะบอกการcomplieและวิธีการติดตั้ง ก่อนที่เว็บจะใช้งาน หรือแม้กระทั่ง การรันserver(execution))
- A "Resources" section listing classic references related to the topic (documentation, articles, tutorials, etc.), as well as a description of how AI was used - specifying for which tasks and which parts of the project. (แหล่งที่เกี่ยวข้อง เช่น บทความ, เว็บguide ต่างๆที่เราไปดูมาตอนศึกษา tool ต่างๆ, เหมือนเคยอธิบายด้วยว่าเอา AI ไปใช้ในงานอะไรบ้าง ในส่วนต่างๆของโปรเจค)
- Additional sections may be required depending on the project (e.g., usage example, feature list, technical choices, etc.). ()
