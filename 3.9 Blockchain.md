# IV.9 Blockchain
Major = 2 points, Minor = 1 point
- Major: Store tournament scores on the Blockchain (เก็บคะแนนการแข่งขันแบบ tournament บน Blockchain)
  - Use Avalanche and Solidity smart contracts on a test blockchain. (ใช้ Avalanche และ Solidity บน test blockchain)
  - Implement smart contracts to record, manage, and retrieve tournament scores. (นำระบบมาบันทึก, จัดการ และดูคะแนนการแข่งขัน)
  - Ensure data integrity and immutability. (ข้อมูลมีความถูกต้องน่าเชื่อถือ และ เปลี่ยนแปลงไม่ได้)
- Minor: Use ICP (Internet Computer Protocol) for a backend that runs on a blockchain (uncompatible with SSR). (ใช้ ICP ทำระบบ Backend บน Blockchain โดยห้าม/ไม่รองรับการใช้ SSR(Server-side Rendering))

- ## ICP = เป็นเครือข่ายบล็อกเชน (Blockchain) ยุคใหม่ที่ถูกออกแบบมาให้สามารถ รันเซิร์ฟเวอร์/ระบบ Backend ทั้งหมดไว้บน บล็อกเชน ได้เลย (ต่างจาก Bitcoin หรือ Ethereum ที่ทำได้แค่รัน Smart Contract เล็กๆ) สิ่งที่รันบน ICP จะถูกเรียกว่า Canister (เทียบเท่ากับ Docker Container บนบล็อกเชน)
- ## SSR = เทคนิคการทำเว็บที่ เซิร์ฟเวอร์ช่วยประกอบร่าง HTML แล้วส่งกลับมาให้เบราว์เซอร์ (เช่น Next.js แบบปกติ หรือ Django/Laravel ที่ส่ง Rendered HTML)
